# v0 Handoff

Prepare a v0 prompt from the current technical state.

Do not ask v0 to invent backend behavior. Include:

- exact route/page to design
- existing components and files
- data already available
- actions already wired
- actions not wired yet
- copy constraints
- visual constraints
- what not to build

HireWire design rules:

- Build the actual product screen, not a marketing page.
- Hide machinery.
- Use Prove Fit and Match Interview language.
- Avoid evidence picker/database/manual mapping language in primary flows.
- Keep readiness and apply gate honest.
- Do not show CTAs that are not wired unless explicitly marked as disabled.

Output:

1. v0 prompt
2. files v0 should respect
3. backend assumptions v0 must not make
4. acceptance criteria
