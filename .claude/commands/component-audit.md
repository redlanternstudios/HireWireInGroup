# Component Audit

Audit components. Do not edit files.

Inspect:

- `components/`
- dashboard pages using local components
- route-local components under `app/`

Create a table:

- component
- file path
- used on
- purpose
- real data or mock
- props
- state
- handlers
- duplicate similar components
- status
- recommendation

Statuses:

- Keep
- Keep but Improve
- Merge
- Remove
- Rebuild
- Dead Code
- Needs Investigation

Pay special attention to dashboard, job cards, job detail, job analysis, evidence matching, evidence library, coach, resume builder, application package, preview, readiness, apply CTA, analytics, navigation, auth, forms, modals, dialogs, and toasts.
