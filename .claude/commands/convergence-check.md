# Convergence Check

Check whether the repo is converging toward the product promise. Do not edit files.

For HireWire, the convergence point is:

```txt
sign up/sign in
-> dashboard
-> add/capture job
-> analyze job
-> compare requirements to Career Context
-> identify weak/missing evidence
-> Match Interview clarifies ambiguity
-> confirmed/skipped proof is saved
-> readiness recalculates
-> Application Package generates from proof
-> package preview shows provenance
-> Ready to Apply gate blocks/allows
-> apply or override is logged
-> outcomes feed back into Career Context
```

For other repos, first identify the equivalent core journey from README, docs, routes, and product copy.

Output:

A. Core journey as implemented
B. Core journey as intended
C. Where the journey breaks
D. Features that are visually present but not wired
E. Duplicate systems or competing sources of truth
F. Data that does not flow downstream
G. P0 convergence blockers
H. Recommended next Build Day scope
I. Recommended next v0 prompt, if UI is the blocker
J. Recommended next Supabase check, if data is the blocker
