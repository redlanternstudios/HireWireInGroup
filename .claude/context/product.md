# HireWire Product Context

HireWire is an Application Readiness Engine.

The primary product promise:

```txt
Upload background -> add job -> prove fit -> generate grounded package -> pass readiness gate -> apply -> learn from outcome
```

The user should feel guided, not managed.

## Core Flow

1. Sign up or sign in.
2. Dashboard.
3. Add or capture job.
4. Analyze job post.
5. Compare job requirements against user career context.
6. Identify missing or weak evidence.
7. Use coach conversation to clarify evidence.
8. Build application package.
9. Preview application package.
10. Pass readiness gate.
11. Apply or log override.
12. Track status and outcomes.
13. Feed outcomes back into career context.

## Product Language

Use:

- Prove Fit
- Match Interview
- Career Context
- Application Package
- Ready to Apply
- proof
- source
- confirmed
- skipped

Avoid in primary user flows:

- database
- evidence picker
- manual mapping
- requirement table
- manage evidence
- workflow theory
- step counters

## Design Principle

Hide the machinery.

The user should only see the moment where their judgment is needed. Everything else should be automatic, invisible, or summarized as proof.
