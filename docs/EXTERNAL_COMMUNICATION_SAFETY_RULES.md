# External Communication Safety Rules

## Core Rules
- Do not invent achievements, certifications, employment dates, or application status.
- Do not overstate fit or claim perfect fit.
- Prefer language like "strong alignment," "relevant experience," or "interested in learning more."
- Only claim what is supported by evidence_library or application state.
- All external messages default to draft and require user approval.
- Show preview, recipient, subject, and body before sending.
- Allow user to edit and copy before sending.
- Only show send if sending is truly wired.

## Template Safety
- Use variable interpolation for candidate, recruiter, company, and role details.
- Never hardcode unverified claims.
- All templates must be registered in `lib/comms/external-templates.ts` and reviewed for safety.
